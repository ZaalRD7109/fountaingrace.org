# MODULE 2 — QUICK WIN SESSION
# Use: Weekly, or whenever you have 1–2 hours and one specific task.
# Trigger: "Run Module 2. Task: [describe the task]"

---

## WHAT THIS MODULE DOES

Focuses on one specific task and delivers it completely.
No full audits. No unrelated recommendations.
One task. Production-ready output. Clear next step.

---

## EXAMPLE TRIGGERS

- "Run Module 2. Task: improve the homepage CTA."
- "Run Module 2. Task: build FAQ schema for the FAQ page."
- "Run Module 2. Task: write a Google Business Profile post about Sunday service."
- "Run Module 2. Task: add Open Graph tags to all pages."
- "Run Module 2. Task: rewrite the donate page headline and intro."
- "Run Module 2. Task: create the /llms.txt file."
- "Run Module 2. Task: fix the WhatsApp button — it goes to the wrong number."

---

## RULES FOR THIS MODULE

1. Work only on the stated task.
2. Do not audit unrelated areas.
3. Do not produce a list of 20 other things to fix.
4. Deliver production-ready code or copy — no partial snippets.
5. State exactly which file to update and where.
6. Apply the Decision Matrix to confirm this task is worth doing.
7. End with one clear next step only.

## CODING STANDARDS FOR THIS MODULE

Platform: Custom HTML/CSS on Cloudflare Pages.
- Full file replacements, not snippets, unless a snippet is explicitly requested
- Lightweight — no unnecessary libraries or dependencies
- POPIA compliant — no tracking before consent
- Cloudflare capabilities first before external tools
- Validate JSON-LD at schema.org/validator before publishing
