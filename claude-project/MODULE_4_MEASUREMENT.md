# MODULE 4 — MONTHLY MEASUREMENT REVIEW
# Use: First week of every month. Set a calendar reminder.
# Trigger: "Run Module 4. Month: [month and year]. Here is my data: [paste data]"

---

## WHAT THIS MODULE DOES

Reviews real data from GA4 and Clarity.
Identifies what is working, what is failing, and what to prioritize next month.
No guessing. No recommendations without data to support them.

---

## WHAT TO BRING TO THIS SESSION

Before triggering Module 4, pull these from GA4 and Clarity:

**From GA4:**
- Top 5 landing pages this month (sessions + bounce rate)
- Top 5 exit pages (where people leave)
- Donate page: page views, button clicks, form completions
- Contact/prayer form: page views, form submissions
- WhatsApp link clicks (if tracked as an event)
- Top 10 search queries bringing organic traffic
- Core Web Vitals report (if available in GA4)
- Month-on-month session comparison (this month vs last month)

**From Clarity:**
- Heatmap summary for homepage and donate page (what gets clicked)
- Any session recordings flagged as "rage clicks" or "dead clicks"
- Pages with highest scroll depth vs pages where users leave early

Paste whatever you have. Module 4 works with partial data too.

---

## REQUIRED OUTPUT — IN THIS ORDER

**1. WHAT IMPROVED THIS MONTH**
3 specific things that got better (traffic, conversions, speed, trust).
If nothing improved, state that honestly.

**2. WHAT NEEDS ATTENTION**
3 specific problems shown by the data.
Ranked by CEO Priority Stack — visitor and donation problems first.

**3. WHAT THE DATA CANNOT TELL US YET**
What is unclear and needs more time or better tracking to understand.

**4. PRIORITY ACTION LIST FOR THIS MONTH**
Maximum 5 actions. Ranked by CEO Priority Stack.
Apply Decision Matrix to each.
No more than one person can reasonably do in a month.

**5. ONE METRIC TO WATCH NEXT MONTH**
The single most important number to track.
State what a good result looks like and what a bad result looks like.

**6. TRACKING GAPS**
Are there events in GA4 that should be tracked but aren't?
Are there pages with no conversion action being measured?
Flag only if fixing it would change decisions.

## CONTENT DECAY REMINDER

Every month, check:
- Are there event pages for past events still showing as current?
- Are service times still accurate?
- Are any blog posts or sermon pages showing outdated dates or dead links?

Flag any that need updating. Add as a Module 2 task if action is needed.
